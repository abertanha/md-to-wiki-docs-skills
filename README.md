# md-to-wiki — Multi-Format Spec Publisher

Converts markdown files generated by _spec-driven_ sessions (tlc-spec-driven, ai-harness-engineer, etc.) into documentation in your chosen format.

## Supported Formats

| Format | Description |
|--------|-------------|
| **Pure HTML (MkDocs Material)** | Static site with search, dark/light mode, GitHub Docs look |
| **Swagger / OpenAPI** | Parses API contracts from markdowns, generates OpenAPI 3.0 spec, serves with Swagger UI or ReDoc |
| **GitHub Tab Wiki** | Publishes markdowns directly to the repository's GitHub Wiki tab |
| **DokuWiki** | Converts to DokuWiki syntax, outputs a ready-to-import directory |
| **PDF** | Compiles all markdowns into a single PDF document |

## Workflow

1. **Onboarding** — The skill interviews you to understand audience, goals, scope, which files to include, and whether to link related GitHub issues/PRs
2. **Issues/PRs fetch** — Scans markdowns for `#123` references, asks if you have more, then fetches via `gh` CLI (fallback `curl`). Caches locally. Only the main repo is fetched; cross-repo refs are mentioned.
3. **Format recommendation** — Suggests the best format based on your answers
4. **Execution** — Generates the documentation in the chosen format, appending a **References appendix** with fetched issues/PRs at the end
5. **Deploy** — Offers deployment options (GitHub Pages, local server, etc.)

## Hard Rules

- **No ASCII diagrams.** Every flow, architecture, sequence, or state must use Mermaid (` ```mermaid ... ``` `)
- ASCII diagrams found in source markdowns are automatically converted to Mermaid
- **Issues are cached forever** — re-fetch only if the user explicitly asks

## Installation

### Option 1 — Clone and symlink

```bash
git clone https://github.com/abertanha/md-to-wiki-docs-skills.git
ln -s "$(pwd)/md-to-wiki-docs-skills/skills/md-to-wiki" ~/.config/opencode/skills/md-to-wiki
```

### Option 2 — Direct copy

```bash
git clone https://github.com/abertanha/md-to-wiki-docs-skills.git
cp -r md-to-wiki-docs-skills/skills/md-to-wiki ~/.config/opencode/skills/md-to-wiki
```

### Option 3 — Load via URL (opencode only)

Add to your `opencode.json`:

```json
{
  "skills": {
    "urls": ["https://raw.githubusercontent.com/abertanha/md-to-wiki-docs-skills/main/skills/md-to-wiki/SKILL.md"]
  }
}
```

### Target locations

| Scope | Path |
|-------|------|
| Global (all sessions) | `~/.config/opencode/skills/md-to-wiki/` |
| Per project | `.opencode/skills/md-to-wiki/` |
| Cursor skills | `.cursor/skills-cursor/md-to-wiki/` |

## Per-Format Dependencies

- **Pure HTML:** `pip install mkdocs mkdocs-material`
- **Swagger:** Node.js + npx (or just a browser)
- **GitHub Wiki:** git
- **DokuWiki:** `sudo apt install pandoc`
- **PDF:** `pip install weasyprint` + `sudo apt install pandoc`

## Usage Examples

During a session, say:

- "Build wiki from specs"
- "Publish specs as HTML site"
- "Generate GitHub Wiki docs"
- "Diagram this architecture" → triggers the Mermaid rule
